Under the Hood of Amazon Cloud Computing

Amazon Elastic compute cloud or EC2 is a key part of Amazon.com’s cloud computing platform and Amazon Web Services. EC2 allows users to rent virtual computers on which to run their own computer applications. After analyzing the EC2 infrastructure, it was found to have 454,400 servers.

Cloud Servers

EC2 allows scalable deployment of applications through which a user can create a virtual machine, which Amazon calls an “instance”. A user can create, launch, and terminate server instances as needed, paying by the hour for active servers. This is why it has been coined the name “elastic”.

Elastic Compute Units and Block Storage

Elastic compute unit (ECU) provides the equivalent CPU capacity of a 1.0-1.2 GHz 2007 Opteron or 2007 Xeon processor. The elastic block store (EBS) provides raw block devices for EC2 instances. EBS volumes can be up to 1 terabyte in size. They are built on replicated storage so that the failure of a single component will not cause data loss.

Server Types

EC2 utilizes Xen virtualization where each “instance” functions as a virtual private server. As of December 2012, the following instance types were offered:

  • On-demand: pay by the hour without commitment
  • Reserved: rent instances with one-time payment
  • Spot: bid-based service

Operating System

The EC2 service offered Linux and later Sun Microsystems’ OpenSolaris and Solaris Express Community Edition. It also added Windows Server 2003 and Windows Server 2008 operating systems to the list. Amazon also added their own Linux distribution based on the Fedora and Red Hat Linux. This included:

  • Linux Kernel
  • Java OpenJDK Runtime Environment
  • GNU Compiler Collection

Storage

EBS volumes provide persistent storage independent of the lifetime of the EC2 instance and act much like hard drives on a real server. The OS is free to use the device however it wants. Users can set up and manage storage volumes of sizes from 1 GB to 1 TB. Amazon's elastic IP address feature is similar to static IP address in traditial data centers. There is one key difference however. A user can programmatically map an elastic IP to any virtual machine instance without a network administrator's help and without having to wait for DNS to propagate the new binding.
